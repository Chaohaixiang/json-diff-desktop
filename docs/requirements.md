## JSON Diff 桌面应用 - 需求文档

### 一、产品定位

- **目标用户**：
  - 经常需要对比 JSON 的前端/后端工程师、测试工程师。
  - 希望在本地快速对比接口返回、配置文件、调试数据的技术人员。
- **使用场景**：
  - 对比两次接口返回的差异（线上 vs 线下、旧版本 vs 新版本）。
  - 对比配置变更，快速排查线上故障。
  - 在没有网络或不方便上传敏感数据时，本地安全对比 JSON。

### 二、核心功能需求

1. **双列 JSON 编辑器**
   - 页面上方左右两列分别显示 JSON A 和 JSON B。
   - 支持粘贴、编辑、清空。
   - 支持滚动且不联动（用户可自由对齐对比）。

2. **差异分析**
   - 在任意一侧 JSON 变更后，自动节流（约 300ms）重新计算 diff。
   - 支持三种变更类型：
     - 新增（added）
     - 删除（removed）
     - 修改（modified）
   - 支持嵌套对象与数组，路径以点号/下标形式表示，例如：
     - `user.name`
     - `projects[0].status`

3. **可视化高亮**
   - 在上方编辑区中：
     - 新增行/值使用绿色系底色与高亮文字。
     - 删除行/值使用红色系底色与高亮文字。
     - 修改行/值使用黄色系底色与高亮文字。
   - 下方结果列表中：
     - 清晰展示 JSON 路径、类型、值变化（`旧值 → 新值`）。
     - 类型使用彩色标签（新增/删除/修改）。

4. **结果列表视图**
   - 当有差异时：
     - 列出全部差异，支持滚动浏览。
     - 支持按类型 + 路径排序（修改优先，其次新增、删除）。
   - 当无差异时：
     - 显示友好提示「两份 JSON 完全一致」。
   - 当输入为空或不完整时：
     - 显示「请粘贴两份 JSON 后自动对比」或「请在左右两侧都粘贴 JSON 后再进行比对」。
   - 当 JSON 格式错误时：
     - 结果面板顶部显示错误提示，不影响上方继续编辑。

5. **格式化与语言/主题切换**
   - 顶部工具栏：
     - 「格式化」按钮：同时美化 JSON A/B 的缩进与格式。
     - 「比对」标签：当前页面状态展示（非按钮）。
     - 语言切换：`中 / EN` 双语切换。
     - 主题切换：明/暗两个主题。

6. **本地桌面体验**
   - 通过 Tauri 将前端打包为 Windows 桌面应用。
   - 支持自适应窗口大小，限制最小宽高避免布局挤压。
   - 支持通过桌面快捷方式启动应用（无需命令行）。

### 三、非功能性需求

1. **安全性**
   - 所有 diff 计算在本地浏览器/应用内完成，不上传任何数据。
   - 不包含远程同步或云端存储逻辑。

2. **性能**
   - 针对中等大小的 JSON（约几万行）应保持可用，不出现明显卡顿。
   - 使用节流与增量更新避免每次按键都触发重计算。

3. **可用性与体验**
   - 即使只有一侧 JSON 时也要给出清晰提示，而不是空白或错误。
   - 错误信息以友好文案提示，而不是技术堆栈。
   - 高亮颜色在深色和浅色主题下都保持足够的对比度。

4. **可维护性**
   - 前端使用 TypeScript，保证类型安全。
   - Diff 逻辑与 UI 组件分离，便于未来替换 diff 引擎或扩展功能。

### 四、后续可扩展方向（非必须）

- 支持将 diff 结果导出为 JSON/Markdown 报告。
~- 支持从文件选择器加载 JSON 文件，或从 URL 拉取 JSON。~
~- 支持「忽略字段」或「只看指定路径」的过滤功能。~
~- 支持 Diff 历史记录与多 tab 管理。~

（带删除线的为暂缓需求，当前版本不实现，只在文档中记录以便后续迭代。）
